4. Mention two reasons why branches are used in a Git repository (4 points)
	Branches allow developers to work on new features or fixes without affecting the main branch.
	Branches make it easier to test, review, and merge changes safely.

5. What is the difference between git log and git status? (4 points)
	git log shows the commit history of the repository.
	git status shows the current state of the working directory and staging area, including modified, staged, and untracked files.

8. In the context of object-oriented programming (I) What is inheritance? (II) What is polymorphism? (III) What is encapsulation? (6 points)
	(I) Inheritance is when one class acquires the attributes and methods of another class, allowing code reuse and the creation of hierarchical relationships between classes.
	(II) Polymorphism is the ability for different classes to be treated through a common interface, where the same method name can behave differently depending on the object.
	(III) Encapsulation is the practice of bundling data and methods together inside a class and restricting direct access to internal details.

9.  What is the main difference between the “Dictator and Lieutenants” workflow and the “Integration manager” workflow? (5 points)
	The main difference is that in Dictator and Lieutenants workflow, trusted lieutenants collect and integrate changes from groups of developers before passing them to a central leader or "dictator" who makes the final integration decisions. In the Integration manager workflow, each developer typically works in their own fork, and one integration manager reviews and pulls approved changes into the main repository. 

10. How would a team of 100 developers benefit from following the “Dictator and Lieutenants” workflow instead of the “Centralized” workflow? (5 points)
	A team of 100 developers would benefit because the Dictator and Lieutenants workflow scales better. It reduces bottlenecks by dividing responsibility among lieutenants, improves code review and coordination, and prevents all developers from pushing directly into one central branch. This makes large projects easier to manage and helps maintain code quality.
